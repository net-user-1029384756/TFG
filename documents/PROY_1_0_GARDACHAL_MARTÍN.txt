3. INTRODUCCIÓN
--------------------------------------------------------------------------------
La administración de infraestructuras de red es crítica en organizaciones modernas que enfrentan entornos multi-fabricante y necesidades de escalabilidad[cite: 15, 16]. La automatización reduce errores humanos y garantiza la homogeneidad de la infraestructura[cite: 17, 244]. 

Este proyecto desarrolla una solución modular para gestionar routers y switches (Cisco, TP-Link, Mikrotik) y preparar equipos cliente (Windows/Linux) mediante Python (POO) y Ansible[cite: 18, 19, 20]. Se validará en entornos reales o simuladores profesionales como GNS3 o EVE-NG[cite: 24, 251].

4. CONTEXTUALIZACIÓN
--------------------------------------------------------------------------------
- Ámbito Social: Responde a la necesidad de redes estables para teletrabajo y servicios digitales[cite: 257, 258].
- Ámbito Económico: Ahorro en horas de configuración y uso de herramientas de código abierto como Ansible[cite: 260, 262].
- Ámbito Tecnológico: Uso de protocolos OSPF, CDP/LLDP y librerías Python como Netmiko y NAPALM[cite: 264].
- Ámbito Legal: Cumplimiento de RGPD mediante conexiones seguras (SSH, WinRM)[cite: 267, 268].
- Ámbito Ambiental: Reducción de desplazamientos físicos y consumo energético mediante virtualización[cite: 270, 271].

5. OBJETIVOS DEL PROYECTO
--------------------------------------------------------------------------------
Objetivo General:
Desarrollar una solución integral de automatización para configurar racks de red y equipos cliente con Python y Ansible, garantizando eficiencia y seguridad[cite: 29, 279].

Objetivos Específicos:
- Diseñar un sistema modular en Python basado en POO[cite: 31, 282].
- Automatizar VLANs, OSPF y seguridad de puertos en dispositivos multi-marca[cite: 32, 283].
- Implementar servicios críticos (DHCP/DNS) mediante Ansible[cite: 33, 284].
- Detectar automáticamente el SO de los equipos clientes y aplicar configuraciones personalizadas[cite: 34, 285].
- Crear una interfaz gráfica (Tkinter/PyQt) para la gestión del sistema[cite: 39, 290].

6. REQUISITOS PREVIOS
--------------------------------------------------------------------------------
Recursos Hardware:
- CPU: Mínimo 4 núcleos / 8 hilos con soporte de virtualización (Intel VT-x/AMD-V)[cite: 48, 49].
- RAM: Mínimo 16 GB DDR4/DDR5 para soportar simultáneamente routers Mikrotik y clientes Windows[cite: 50, 51].
- Almacenamiento: SSD NVMe para soportar altas tasas de IOPS en la simulación[cite: 52, 53].

Recursos Software:
- Hipervisor: Oracle VirtualBox y GNS3[cite: 59].
- OS Host: Windows 11 Pro[cite: 60].
- Dispositivos de Red: MikroTik RouterOS (CHR 7.21) y Cisco IOSv[cite: 61, 62].
- Desarrollo: Python 3.14.0 y Ansible Core operando desde WSL o nodo Linux[cite: 64, 66].
